# Commit Rules

**feat** : Adding a new feature

**fix** : Fixing a bug

**docs** : Changes related to documentation only

**style** : Code changes that don’t affect the logic or functionality (e.g., formatting, white-space, missing semicolons)

**refactor** : Code changes that improve structure or readability but don’t fix bugs or add new features

**perf** : Code changes that improve performance

**test** : Adding or updating tests

**chore** : Updates to the build process, tools, or libraries (e.g., documentation generation, dependency updates)
